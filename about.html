Week 4 - What is a browser? How do they work?

According to Mozilla a web browser is something that that takes you anywhere on the internet. This means that it will take you to any available place on the internet such as websites and search engines. A web browser gets information from these parts on the internet and displays it on the device you are using. This information is sent using http (Hypertext Transfer Protocol) which is used to tell the device and browser how text, videos and other forms of media are transmitted on the internet. (Mozilla, n.d.)

Examples of web browsers include google chrome, internet explorer and opera. These browsers are used by devices to input information and search the internet for that information to display to the user. All browsers do not use and interpret information the same and so this means that on different browsers websites will look and function differently.

Browsers use software called rendering engines to translate the data they receive from the internet into images and text. Hypertext Markup Language (HTML) is the language that the data is written in, and the rendering engines reads this code to provide us with the things we experience on the internet.

The main job of browsers consists of DNS resolution, HTTP exchange, Rendering and repeat. DNs resolution is the process where one enters a URL, and the browser will know which server need to connect to. The browser talks to this server in order to get and IP address so that the browser can find what the URL wants to display.

HTTP exchange happens when our browser communicates with servers in order to get the information we are looking for. Is an exchange of a request from out browser and a reply from the server where we want the data.

Rendering is the process using the data we receive to render the text style images and visuals we see on whatever website we are looking for. Its is the combination of Html, CSS and JavaScript that creates and interactable website for use to see on our devices. (freeCodeCamp, 2018)

Week 6 – The uses of Metadata – what, why, how

Metadata is often referred to as data that describes other data, its known as structured reference data that helps to sort and find out about the information is describes. The word “Meta is a prefix that basically means “and underlying definition or description” which mean that metadata is information that tell us more about the data we are looking at. This can make it easier to find and use instances of data and to see where it was found and use. (Kranz, n.d.)

Why do we use metadata? We use metadata because it helps to improve the data we are using. The metadata makes up for the mistakes people make as people often lose or misplace data, so metadata is way to find the data we were using, and it makes sure we preserve and can re-use this data in the future. It also helps to show the credibility of the data we are using by showing when it was created or edited, and it also shown who created or edited it. (UNC, n.d.)

Metadata is created anytime a document is modified or even deleted this is used to prolong and find data wherever we need it, and it will always make sure that the data we are using comes from a source and we can identify where or who that source is. It is useful as it basically shows us the history of any data we use and can help us if we ever need to verify data or find this data to use. (Kranz, n.d.)
